Entwurf der Architektur

Seit 2016 arbeitet plentymarkets an zusätzlichen Schnittstellen, um die eigene Software für Erweiterungen zu öffnen und für externe Entwickler zugänglich zu machen. Auf Basis dieser Schnittstellen wurden die Plugins Ceres und IO entwickelt, die gemeinsam das neue Standard-Design für plentymarkets Webshops darstellen.

Das Plugin IO umfasst dabei die serverseitige Logik und wurde ursprünglich in der Sprache Hack, später aber in php7 entwickelt. (zu GitHub)

Darauf aufbauend liefert das Plugin Ceres das eigentliche Shop-Frontend aus. Dieses Projekt wurde auf Bootstrap 4 als UI-Framework sowie auf Vue.js als JavaScript-Framework aufgebaut. (zu GitHub)

Einführung in Twig und Vue.js (Vortrag* auf dem Online-Händler-Kongress 2017)

*) Inhalte zum Thema SASS von Jan Mense, Frontend Studios

Überführung bestehender Inhalte nach AsciiDoc

Im ersten Schritt des Projekts mussten bestehende Inhalte des Online-Handbuchs aus dem Content-Management-System von plentymarktes exportiert und aus dem vorliegenden Format (HTML mit Template-Logik) nach AsciiDoc überführt werden, um diese Dokumente mit GIT verwalten zu können.
Im nächsten Schritt wurde ein Prozess geschaffen, um diese Dateien in ein entsprechendes Plugin zu überführen, um auf diesem Weg wieder eine darstellbare Online-Version der Inhalte zu erhalten.

Beide Prozesse wurden mit Node.js realisiert.

Entwicklung wiederverwendbarer Angular-Komponenten

Um eine Erweiterungen der eigenen Software auch im Bereich des Backends einfach zu ermöglichen, stellt plentymarkets eine Sammlung wiederverwendbarer Angular-Komponenten für die Entwicklung von Erweiterungen zur Verfügung. Dabei steht die Wiederverwendbarkeit der Komponenten für verschiedenste Anwendungsfälle besonders im Fokus.
